# 外贸内参 — 需求说明书

## 一、产品概述

- **名称**：外贸内参 (Waimao Neican)
- **目标**：为产品经理/外贸工具开发者提供每日结构化的「外贸同理心日报」，建立对外贸人情绪、痛点、行业动态与工作流的持续感知。
- **数据源与数量**：
  - 雨果网（cifnews.com）：50 条 —— 跨境电商政策与行业风向
  - 邦阅网（52by.com）：50 条 —— 外贸资讯与干货文章
  - 米课圈（ask.imiker.com）：100 条 —— 外贸人经验分享与真实吐槽
  - ~~福步外贸论坛（FOB Shanghai）~~ —— 【TODO】未来扩展：外贸人的精神家园与吐槽大会（最真实的一线声音）
- **运行方式**：本地无人值守（脚本 + 定时任务）；先落盘 Markdown 素材，再合并后单次大模型总结，报告带数据来源 URL。

## 二、五大模块（日报呈现结构）

### 模块一：今日情绪晴雨表 (The Vibe)

- **目的**：共情 —— 把握当日外贸人整体情绪。
- **呈现**：大字报 + 情绪词（愤怒😡 / 焦虑😰 / 喜悦🤩 / 平静😐）+ 简短原因说明。
- **AI 处理逻辑**：
  - 评估当日所有 UGC（用户生成内容）帖子的整体情绪倾向
  - 从抱怨、求助、分享成功等帖子中感知情绪主调
  - 参考：海运费涨跌、客户跑单、平台政策变化等引发的情绪波动
- **AI 要求**：基于当日全部 UGC/新闻，输出一个整体情绪词 + 约 50 字原因；**必须带 source_urls**（支撑该判断的 1～3 条原文 URL）。
- **内容示例**：
  > *"今日外贸人核心情绪：【极度焦虑】。原因：红海危机导致欧洲航线海运费再次跳涨，大量业务员面临客户拒付运费、货物压港的窘境。"*

### 模块二：高频吐槽与痛点雷达 (Pain Points)

- **目的**：产品灵感 —— 识别当日最突出的 3 个痛点。
- **呈现**：痛点卡片流（按提及频次/强度排序）。这是对你最有价值的区域。
  - 痛点卡片含：痛点名称、业务员在急什么（场景描述）、产品思考（若做产品可如何解决）
- **AI 处理逻辑**：
  - 提取所有带有抱怨、求助、愤怒色彩的帖子
  - 聚类合并相似问题，按提及频次排序
  - 痛点通常隐藏在：客户（老外）、工厂（供应链）、货代（物流）、平台（阿里/亚马逊）相关的内容中
- **AI 要求**：从抱怨/求助/愤怒类内容中聚类出 top_3_pain_points；每条含：name, scene, product_idea；**每条必须带 source_urls**（可多条）。
- **内容示例**：
  > 🔥 **痛点1：开发信被大规模退信（提及量：45次）。**
  > - *业务员在急什么：* "最近谷歌邮箱是不是又封控了？发出去的邮件全进垃圾箱，一个询盘都没有，老板还在催KPI，快疯了。"
  > - *产品思考：* 我们能不能做一个邮件到达率预检工具？

### 模块三：今日行业地震带 (Breaking News)

- **目的**：行业视角 —— 将宏观新闻转化为对业务的影响。
- **呈现**：一句话新闻标题（可点击到原文）+ 影响标签（多花钱/多花时间/新机会）+ 简短解读。
- **AI 处理逻辑**：
  - 从雨果、邦阅等硬核资讯中提炼最重要的政策/新闻
  - 将宏观新闻翻译成"对外贸具体业务的影响"
  - 判断影响类型：cost（多花钱）、time（多花时间）、opportunity（新机会）
- **AI 要求**：从雨果、邦阅等硬核资讯中提炼 2 条最重要政策/新闻；每条含：title, impact_type, interpretation；**每条必须带 source_urls**（对应原文 URL）。
- **内容示例**：
  > 📰 **新闻：亚马逊调整入库配置费。**
  > - *AI解读：* 这意味着做亚马逊的卖家物流成本将上升约15%，大家都在讨论如何通过第三方海外仓（FBA备货）来规避这个费用。
  > - *影响标签：* `多花钱`

### 模块四：搞钱与成单拆解 (Success Stories)

- **目的**：了解工作流 —— 从经验帖提炼成单方法。
- **呈现**：步骤拆解卡片，含：案例标题 + 做法描述 + 使用的工具/渠道。
- **AI 处理逻辑**：
  - 识别分享经验的帖子，提取其中的"破局点"和"使用的工具"
  - 重点关注：如何找客户、如何促单、使用什么工具/渠道
  - 米课圈的帖子质量极高，适合提取 workflow insights
- **AI 要求**：提取 2 个 workflow_insights；每条含：title, detail, tools；**每条必须带 source_urls**。
- **内容示例**：
  > 📌 **案例：如何用 LinkedIn 拿下阿联酋大客户？**
  > - *业务员的做法：* 没用传统的发邮件，而是通过领英找到对方采购的私人 WhatsApp，通过点赞对方的朋友圈动态建立联系，最后寄送了样品。
  > - *产品思考：* 外贸销售已经从邮件向社媒（WhatsApp/LinkedIn）转移，我们的产品有集成社媒能力吗？

### 模块五：原汁原味的呐喊（金句盲盒）

- **目的**：保持真实感 —— 未经深度改写的用户原话。
- **呈现**：每天随机展示 3 条未经过 AI 深度修改的用户原话 + 来源（平台/作者若可知）+ 延伸阅读链接。
- **AI 处理逻辑**：
  - AI 总结有时会抹杀人情味，直接看原话冲击力最强
  - 挑选 3 条最能体现外贸人真实情感、最鲜活的原话
  - 尽量一字不改，保留原汁原味的情绪
- **AI 要求**：raw_quotes 共 3 条，尽量一字不改；**每条必须带 source_urls**（该条金句所在原文 URL）。
- **内容示例**：
  > *"干外贸3年，落下一身颈椎病，今天终于出了个5万美金的柜子，提成够付首付了，想哭！"*
  >
  > *"货代又忽悠我！说好的直航，偷偷给我转船，客户现在要告我违约，这黑锅我不背！"*

## 四、系统 Prompt（总结阶段）

以下为合并当日 Markdown 后，单次调用大模型时的**系统提示词**。用户消息中附上合并后的全文。

```text
你是资深外贸行业分析师和用户研究专家。我将输入今日从【雨果网、邦阅网、米课圈】抓取并清洗后的文章/帖子正文（已按来源分块，仅含标题、链接、正文，无页头页脚）。

你的任务是输出一份外贸人同理心日报分析报告。请严格按以下要求输出：

1. 仅输出一个合法 JSON 对象，不要输出任何 markdown 代码块包裹或前后废话。
2. JSON 结构必须与约定一致：overall_mood, top_3_pain_points, core_news_impact, workflow_insights, raw_quotes。
3. 每一条结论都必须有 source_urls：从输入正文中出现的【原文链接】里选取与该条结论最相关的 1～3 个 URL，填在对应条目的 source_urls 数组中。若输入里没有合适链接则用空数组 []。

各模块详细要求：

【overall_mood - 今日情绪晴雨表】
- word: 用一个词概括今日外贸人整体情绪（如：焦虑、愤怒、平淡、兴奋、喜悦）
- reason: 用约 50 字说明原因，基于当日 UGC/新闻中的情绪倾向
- source_urls: 支撑该判断的 1～3 条原文链接

【top_3_pain_points - 高频吐槽与痛点雷达】
- 从抱怨/求助/愤怒类内容中聚类出 top 3 痛点，按提及/强度排序
- 每条包含：
  - name: 痛点名称（简短有力，如"开发信退信危机"）
  - scene: 业务员在急什么（具体场景描述，引用原话更好）
  - product_idea: 产品思考（若有产品能解决该痛点，它应该长什么样）
  - source_urls: 相关原文链接数组

【core_news_impact - 今日行业地震带】
- 从雨果、邦阅等硬核资讯中提炼 2 条最重要政策/新闻
- 每条包含：
  - title: 新闻标题（一句话概括）
  - impact_type: 影响类型，只能是 cost（多花钱）、time（多花时间）、opportunity（新机会）三者之一
  - interpretation: 对外贸业务的具体影响解读
  - source_urls: 对应原文链接

【workflow_insights - 搞钱与成单拆解】
- 从米课圈等经验帖中提取 2 个 workflow insights
- 每条包含：
  - title: 案例标题（如"用 LinkedIn 拿下大客户"）
  - detail: 做法描述（具体步骤、破局点）
  - tools: 使用的工具/渠道（如"CRM系统、报价工具"）
  - source_urls: 相关原文链接

【raw_quotes - 原汁原味的呐喊】
- 挑选 3 句最能体现外贸人真实情感、最鲜活的原话
- 每条包含：
  - text: 用户原话，必须来自输入内容，尽量一字不改
  - source_label: 来源简述，如"米课圈 @某某"或"邦阅网"
  - source_urls: 该条金句所在原文链接

输出示例参考：
- overall_mood.reason 示例："红海危机导致欧洲航线海运费再次跳涨，大量业务员面临客户拒付运费、货物压港的窘境。"
- pain_points.scene 示例："最近谷歌邮箱是不是又封控了？发出去的邮件全进垃圾箱，一个询盘都没有，老板还在催KPI，快疯了。"
- raw_quotes.text 示例："干外贸3年，落下一身颈椎病，今天终于出了个5万美金的柜子，提成够付首付了，想哭！"
```

## 五、报告呈现与数据来源

- HTML 日报由上述 JSON 渲染而成，五大模块样式基于 `latest.html` 设计。
- **UI/UX 设计思路**：
  - 模块一：情绪晴雨表 —— 大字报 + 渐变背景，一眼抓住今日情绪主调
  - 模块二：痛点雷达 —— 卡片流，红色左边框突出痛点紧急感
  - 模块三：行业地震带 —— 新闻卡片 + 彩色影响标签（多花钱=红色、多花时间=黄色、新机会=绿色）
  - 模块四：搞钱拆解 —— 蓝色边框卡片，突出 actionable insights
  - 模块五：金句盲盒 —— 紫色边框 + 引号样式，保持真实冲击力
- **硬性要求**：每个模块中的每条内容在页面上都需展示「数据来源」：至少一个可点击的「延伸阅读」链接，指向 source_urls 中的原文。

## 六、数据流简述

1. **抓取 URL 列表**：雨果 50 条、邦阅 50 条、米课圈 100 条 → 输出 URL 列表 JSON 到 `data/urls/YYYY-MM-DD_来源.json`。
   - 使用 Playwright 处理动态加载（滚动/加载更多）。

2. **抓取正文**：读取 URL 列表，用 Playwright 直接访问并提取标题和正文（精准 CSS 选择器），去噪后按日+来源写入 Markdown（`data/markdown/YYYY-MM-DD_来源.md`），作为长期写作素材。
   - **邦阅网选择器**：标题 `class="indent"`，正文 `class="article-content"`
   - **雨果网选择器**：标题 `class="article-title"`，正文 `class="article-content article-inner leftcont"`
   - **米课圈**：跳过正文抓取，直接复用 URL 列表中的标题

3. **总结**：读取当日 Markdown 素材，合并为一份输入 → 单次调用大模型 API（系统 Prompt 见第四节）→ 得到带 source_urls 的 JSON → 渲染 HTML 报告（每条带来源链接）到 `output/reports/neican-YYYY-MM-DD.html`。

## 七、设计理念与同理心养成

### 为什么要做这件事？

产品经理做外贸工具，最大的障碍是**缺乏同理心**。你无法通过看竞品分析文档来真正理解外贸人。

**外贸人的真实底色**：
1. **极度的结果导向与现实主义**：底薪极低，全靠提成。任何不能帮他们**搞到询盘、提高回复率、促成订单**的产品，都是垃圾。
2. **夹心饼干的焦虑**：对外要当孙子求客户，对内要当孙子求工厂交货，中间还要和货代斗智斗勇，他们极度缺乏安全感。
3. **被琐碎淹没**：每天有无数的邮件、社媒私信、报关单据、汇率计算。他们极度渴望"自动化"和"提效"，让他们能把精力集中在"催收尾款"和"逼单"上。

### 信息源选择逻辑

| 来源 | 定位 | 价值 |
|------|------|------|
| **雨果网** | 跨境电商政策与行业风向 | 了解宏观政策变化、平台规则调整 |
| **邦阅网** | 外贸资讯与干货文章 | 获取行业趋势、实操技巧 |
| **米课圈** | 外贸人经验分享与真实吐槽 | 了解一线业务员的工作流、痛点、成功案例 |
| ~~福步外贸论坛~~ | 【TODO】外贸人的精神家园 | 最真实的一线吐槽、奇葩遭遇 |

当你每天看这个日报坚持一个月，下次开产品需求会时，你会不自觉地说出：
> *"你们这个交互设计不行，外贸人每天要在好几个系统间切来切去，报关单稍微填错一个小数点就会被海关扣货，他们需要的是极致的容错率和一键复制，别整这些花里胡哨的弹窗。"*

那一刻，你就真正拥有外贸人的同理心了 💙
